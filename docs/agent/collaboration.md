# Agent 多人协作与任务交接规范

## 1. 读取时机

以下场景必须完整读取本文件：

- 主脑创建、拆分或分配 change。
- 执行 Agent 接收或开始任务。
- change 更换负责人、出现阻塞或需要改变范围。
- 执行 Agent 完工并向主脑交接。

本文件说明任务如何在主脑与执行者之间流转；OpenSpec artifact 的具体格式仍以 `openspec/config.yaml` 为准。

## 2. 角色职责

### 主脑 Agent

主脑负责：

- 从最新 `main` 创建与 change 对应的任务分支。
- 探索需求并创建 proposal、design、specs、tasks。
- 明确 change 名称、负责人、分支、依赖、允许范围、禁止范围和验收测试。
- 将大需求拆成可独立交付、可独立验证且文件冲突尽量少的 change。
- 在用户确认后提交并推送规划文档，将远端任务分支交给执行开发者。
- 审查执行结果是否符合规格、测试要求与范围边界。

主脑不得把尚未澄清的产品决策转交执行者自行决定。

### 执行 Agent

执行 Agent 负责：

- 只处理分配给自己的一个 change。
- 读取全部 artifacts，按 tasks 和 TDD 顺序实施。
- 用代码、测试和命令输出作为任务完成证据。
- 对 artifacts 与实现进行最终一致性检查。
- 完成 archive、最终 commit、push、PR 和完工报告。

执行 Agent 不得擅自改变需求含义、成功标准或任务边界。

## 3. change 分配条件

主脑只有在以下条件全部满足时才能交付 change：

- proposal、design、specs、tasks 已齐全并相互一致。
- change 具有唯一负责人和唯一任务分支。
- 依赖的 change、外部条件和执行顺序已经声明。
- Impact 和目录布局明确到文件或目录级别。
- 明确列出禁止修改的相邻模块或高风险区域。
- 每个 Requirement 都有任务覆盖，每组任务可独立验证。
- 测试层级、测试文件位置和最终验收命令已经声明。
- 未决问题已经解决；不得把产品决策伪装成实现细节交给执行者。

任一条件不满足时，change 保持在主脑侧，不得分配。

## 4. 强制开工回执

执行 Agent 修改文件前必须输出：

```text
开发角色：执行 Agent / 主脑 Agent
当前分支：
OpenSpec change：
直接负责人：
已读取文档：
允许修改范围：
明确禁止范围：
依赖与前置条件：
必须执行的测试：
当前阻塞或风险：无 / 具体说明
```

开工回执用于确认 Agent 理解的是同一个任务，不等同于自动获得扩大范围、commit 或 push 的权限。

## 5. 并行协作纪律

- 一个 change 只允许一个直接负责人；其他人只能评审或在主脑重新拆分后接手子 change。
- 不得由多个 Agent 同时勾选同一个 `tasks.md`。
- 不同 change 应尽量避免修改相同核心文件；无法避免时由主脑规定串行顺序。
- 执行者不得从其他分支复制未完成代码来绕开依赖关系。
- 依赖 change 尚未合并时，不得假设其接口或数据结构已经稳定。
- 发现远端分支有他人未知提交时必须停止，确认提交来源和负责人后再继续。
- change 转交时，原负责人必须先提交完整交接报告；未提交代码、临时决定和失败尝试都必须说明。

## 6. 范围变化与阻塞处理

出现下列情况时，执行 Agent 必须停止并报告主脑：

- 需要修改 Impact、design 或 tasks 未覆盖的文件。
- 现有 Requirement 无法实现、相互冲突或存在多种解释。
- 需要新增依赖、迁移数据、改变 API 或公共接口。
- 测试揭示 change 范围之外的缺陷。
- 依赖 change、环境、权限或数据未就绪。
- 实现会破坏其他已归档 spec。

报告格式：

```text
阻塞位置：
对应任务/Requirement：
已验证事实：
需要主脑决定的问题：
可选方案及影响：
当前未执行的操作：
```

主脑决定需要改变范围时，必须先更新 artifacts，再允许执行 Agent 继续。

## 7. 完工与交接报告

执行 Agent 结束任务时必须报告：

```text
OpenSpec change：
完成的任务项：
实际修改文件：
测试命令与结果：
覆盖检查结果：
artifact 一致性检查：
archive 状态：
commit 状态：
push 状态：
PR 状态/链接：
未完成事项：无 / 具体说明
已知风险：无 / 具体说明
建议的后续 change：无 / 具体说明
```

不得用“已完成”“测试正常”等概括性语句替代命令、结果和变更证据。
