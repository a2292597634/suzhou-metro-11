## 目标 / 非目标

**目标：**
- 三个页面各有关键流程 E2E 覆盖
- 用例稳定可重复运行
- 单页加载模式（beforeAll 中加载一次，各测试检查不同元素）

**非目标：**
- 不改业务代码
- 不覆盖跨页面流程（无 PostgreSQL 环境不稳定）
- 不追求 100% 覆盖（视口缩放、Excel 导入等暂缓）

## 单页加载模式

为避免 `page.goto` 重复导航导致的超时问题，采用单页加载模式：

```
beforeAll: page.goto(URL) → 等待关键元素
it('test1'): 检查页面元素（不重新导航）
it('test2'): 检查另一个元素
it('testN'): 交互操作（点击、等待、验证）
```

## 工具

- Puppeteer（已安装）
- server.js（本地 Express 服务器）
- 导航超时 60s（首次），测试超时 60s

## 风险

| 风险 | 缓解 |
|------|------|
| page.goto 频繁超时 | 单页加载模式，避免重复导航 |
| PostgreSQL 不可用导致 API 慢 | 测试不依赖 API 返回内容，只检查 DOM 元素存在 |
| 测试间状态污染 | beforeAll 单次加载，测试按顺序执行 |
