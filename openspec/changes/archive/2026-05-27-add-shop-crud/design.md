## 背景

当前站点编辑模态框中的商铺表格是只读的（除了现有行的输入字段），用户无法新增一行来录入新商铺，也无法删除一行来移除已不存在的商铺。这迫使运营人员必须使用 Excel 导入/导出的方式来间接实现增删，流程繁琐。

## 目标 / 非目标

**目标：**
- 在模态框中支持新增和删除商铺行
- 新增行默认填充合理初始值（空白铺号、默认状态"未出租"）
- 删除时给出确认提示，防止误操作
- 保存时自动重排序号，确保连续性

**非目标：**
- 不修改站点本身的增删（只改商铺）
- 不引入外部 UI 库
- 不做复杂的校验规则（如面积必须为正数等）

## 设计决策

### 1. 新增行插入到表格末尾
**理由**：最简单直观，用户添加完所有商铺后统一编辑。

### 2. 删除按钮放在每行最右侧
**理由**：符合表格操作惯例，不占用主要输入空间。

### 3. 删除后序号由 saveStationEdit 统一重排
**理由**：避免在删除时立即重排导致界面闪烁，统一在保存时处理更稳定。

### 4. 铺号和名称为必填校验
**理由**：这两个字段是识别商铺的最基础信息，其他字段（面积、承租方等）可以留空。

## 风险与权衡

- **风险**：用户误删商铺 → **缓解**：删除前弹出 `confirm` 确认
- **风险**：全部删除导致站点无商铺 → **缓解**：至少保留一行，禁止删除最后一行
